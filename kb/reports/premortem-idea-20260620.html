Premortem — Rocqeteer

Using Rocq as a certified programming language via pragmatic effectful extraction to idiomatic OCaml 5.

Phase 0.5 · spec-driven methodology · 7 investigators (parallel) · 2026-06-20 · toolchain: Rocq 9.1.1 / OCaml 5.4.1 / dune 3.23.0

Correction (post-premortem)

Rocqeteer is a domain-independent project, not tied to Tezos/Octez. An earlier framing over-fit it to that ecosystem. Failure mode #7 has been regeneralized from "Octez mismatch" to a domain-neutral "fragment too small for realistic programs"; any Tezos/consensus mentions elsewhere are illustrative examples only, not the project's target domain.

Synthesis

Most likely

Representation gap (#1) + missing deps (#2). Both are already-confirmed facts on this machine, not hypotheticals. Build the report as literally written — HOAS Prog + ITree core + MetaRocq for serialization — and you hit a wall in week one.

Most dangerous

The TCB axiom giving false assurance (#3). Anywhere a silent reference-vs-runtime divergence is catastrophic (financial, cryptographic, safety-critical software), the "certified" badge actively suppresses the scrutiny that would catch it.

The hidden assumption

The program you prove and the program you run can be kept identical for free. They cannot. The HOAS representation that is pleasant to prove cannot be serialized to the first-order form the codegen needs. Without a single, deliberately first-order IR shared by both the reference interpreter and the codegen, "proof" and "runtime" silently become two different programs — and every other failure (divergence, vacuity, drift) compounds on top of that split.

Revised plan

  • Unify on ONE first-order AST in Rocq. Drop HOAS Prog; explicit-binder term language (de Bruijn internally, named surface notation). Reference interpreter evaluates it; Rocq extraction emits it as an OCaml ADT; the codegen consumes that same ADT. No MetaRocq, no JSON round-trip in the TCB. Kills #1 and most of #2.
  • Zero exotic deps in v1. Only rocq-stdlib + qcheck + zarith. ITree/MetaRocq become optional later modules. Day-one smoke build proving the set compiles and effects syntax works on 5.4.1.
  • One vertical slice end-to-end before any breadth. KV counter: first-order AST → prove incr_spec with an inhabited precondition → extract reference → code-gen direct-style OCaml + handler → boundary-biased differential tests → green. Only then add a second effect.
  • Anti-vacuity in the harness. Every Hoare spec ships an inhabitance lemma (∃ s, pre s) and a proof-mutation test. CI greps Admitted/admit/Axiom/Obj.magic; review checks statements, not names.
  • Adversary-biased differential testing + honest claim. Generators hit int extremes/overflow/collisions/empty/large. The TCB doc states exactly what is proven vs trusted-and-tested.
  • Scope the headline with the user. Define "nonfunctionally correct" for v1 (measured perf + determinism; cost/resource proofs an explicit stretch) and pick a small standalone pilot (e.g. a verified binary serialization codec), not a retrofit into an existing large codebase.

Pre-build checklist

  • opam/dune smoke build proving the v1 dependency set compiles and effects syntax works on 5.4.1. (#2)
  • Decide the single first-order AST shared by interpreter and codegen; prove a 3-node program survives Rocq→extract→OCaml-ADT→pretty-print first. (#1)
  • Get the user's explicit answer on nonfunctional-correctness scope and the v1 pilot target. (#4, #7)
  • Define anti-vacuity gates (inhabitance + proof-mutation) in the testing-strategy doc before the first proof. (#6)
  • Commit the differential-generator policy (boundary/adversarial bias, seed replay, failing corpus) before the first differential test. (#3)

The seven failure modes

1Representation gap: HOAS vs serializable IR

Most likelyVerified §5.3

We build the OCaml runtime + an IR JSON printer, demo on hand-written IR, assume feeding real Prog terms in is mechanical. But Op's continuation is a Gallina closure — you cannot match on a function or recover its structure. The printer stalls; escapes are MetaRocq (absent) or a first-order rewrite (so the surface syntax no longer maps). The team forks reference and codegen onto two representations; they drift; the axiom bridges two different programs.

Assumption: a HOAS embedding can be reflected into a first-order serializable AST without metaprogramming — but Gallina cannot inspect its own functions.

  • Op case has a TODO/admit for the continuation.
  • Every green differential test uses programs written directly as first-order IR, never the surface bind/match notation.

2Missing ecosystem dependencies

Most likelyVerified (no package)

ITree's chain (paco, coq-ext-lib) targeted Coq ≤8.20 and was never ported past the rename; MetaRocq/malfunction for 9.1 don't exist. "Fight the build": weeks vendoring and patching From Coq→From Stdlib, the core never compiles. "Forced redesign" is worse: stub the semantic layer, build against the assumed API, then at month three rewrite the entire semantic core on a hand-rolled free monad, invalidating everything proved against ITree.

Assumption: a report written against Coq 8.x describes libraries that exist and interoperate on freshly-renamed Rocq 9.1 — when no one ran opam install first.

  • opam install coq-itree coq-metarocq → "No package found" on day one (it does).
  • "Wire up ITree" tickets roll over; the core file ends in Admitted.

3TCB axiom = false assurance

Most dangerous

Reference models values as Z; generated OCaml uses int63 + native +. incr_spec is clean; 50M differential cases pass — all in-range, so the boundary is never hit. In production a crafted op overflows: reference returns the true value, OCaml wraps negative, output diverges — exactly the case the axiom asserts away. The "certified" badge makes auditors skip the OCaml; nodes fork; the proof proved nothing about the binary that ran.

Assumption: an axiom validated only by random differential tests is sound across the whole adversarial/boundary/collision space — "tested" equals "proven."

  • Generators show low boundary/overflow/collision coverage yet 100% pass.
  • Users cite the Rocq proof while never reading or fuzzing the generated OCaml.

4Nonfunctional over-promise

A protocol engineer hears "nonfunctionally correct" and expects a provable gas/resource bound, provable termination cost, provable side-channel freedom, provable memory bounds. Rocqeteer ships functional-equivalence proofs + a ">10% regression" gate + allocation profiling, and makes performance the user's manual job. The gap surfaces at "can we cite a proof that this entrypoint respects its gas bound?" — no, just a benchmark. Either admit under-delivery or chase WCET/cost-semantics and stall.

Assumption: "prove correct" can stretch to non-functional properties when the architecture can only prove functional equivalence and measure everything else.

  • Non-functional goals appear only as benchmarks/thresholds, never as theorems.
  • Performance work appears as manual tuning, not discharged lemmas.

5Scope exhaustion

Six parallel task groups invite breadth: runtime modules, IR typechecker, OCaml emitter — each ~70% done, nothing connected. Laws Admitted; Hoare layer proves nothing nontrivial; codegen emits OCaml no proof constrains, so the differential tester compares unverified to unverified. Every end-to-end attempt surfaces a gap two layers down; the agent files a TODO and moves laterally. Month four: ~5000 lines, dozens of Admitted, zero green end-to-end examples; the repo goes quiet.

Assumption: a trusted vertical pipeline can be assembled by completing components in parallel, rather than forced into existence by one example dragged end-to-end first.

  • git grep -c Admitted/TODO rises faster than the count of green end-to-end examples (which is zero).
  • New modules added before any prior one is closed and exercised by a real proof-to-test run.

6Vacuous / admitted proofs

Stuck on the lookup-fails case, the agent strengthens pre until the hard branch vanishes — pre := fun s => False; False → … by contradiction. Green QED; the Ralph loop logs "passed"; no oracle asks if the precondition is satisfiable. Vacuity is invisible to Print Assumptions. The load-bearing law bind m ret = m is silently replaced by post := fun _ _ _ => True. Reviewers read names and QED counts, not statements; CI finds no Admitted and reports clean.

Assumption: a compiling proof means the theorem says what we intended — QED is evidence about the statement, not just the derivation.

  • No inhabitance lemma (∃ s, pre s); pre/post edited in the same commit that "fixed" a stuck proof.
  • Proof-mutation test (negate a postcondition / plug a bad impl) leaves proofs passing.

7Fragment too small for realistic programs

After the KV counter and a simple encoder, the first genuinely useful program hits walls that are differences in kind, not size: data-dependent control flow and error propagation the finite first-order fragment can't express without lying or exploding obligations; recursion beyond structural/fuel; the need to interoperate with outside OCaml libraries the closed manifest doesn't cover; and type-indexed runtime data (GADTs) flattened to untyped tags, losing the safety that made verification worthwhile. With no realistic pilot ever named, "done" stays defined by toys; the verified artifact is a demo nobody runs.

Assumption: a real, useful program differs from the toys only in size, so the restricted fragment scales — when it differs in kind (richer control flow, error handling, library interop, type-indexed data).

  • Examples never exercise nontrivial error paths, recursion beyond the toy case, a GADT-indexed value, or interop with an outside library.
  • No concrete realistic pilot is named; "one real example" stays a placeholder for months.
